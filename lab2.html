# Lab 2: Quick introduction to HDFS

Objective:

To view and manage HDFS files and directories using HDFS shell and NameNode UI.

Successful outcome:

  1. Explore the HDFS Shell
  2. Use the HDFS Shell to manage files and directories in HDFS
  3. Use HDFS Shell command-line options to override the cluster's default HDFS properties
  4. Use the NameNodeUI file browser to view files and directories

# Using the main command hdfs dfs

  1. List the help of the main Hadoop HDFS command:

    hdfs dfs -help

  2. List directory /user in HDFS:

    hdfs dfs -ls /user/

    How is the hierarchy of the HDFS navigation compared to local file systems?

  3. List etc local file system directory using HDFS:

    hdfs dfs -ls file:///etc/

  4. Move labs materials to HDFS using HDFS PUT:

    hdfs dfs -put /home/cloudera/labs/* /user/cloudera/

  5. Give /user/cloudera/ permissions recursively to allow all users to access it:

    hdfs dfs -chmod -R 777 /user/cloudera

  6. Delete the folder /user/cloudera/SentimentFiles/tweets_raw recursively:

    hdfs dfs -rm -r -f /user/cloudera/to_delete/

# Understanding how HDFS stores files and blocks:

  1. Utilize the filesystem check command to look for HDFS information about a file:

    hdfs fsck /user/cloudera/twitter_sentiment/tweets.txt

  2. Generate an HDFS report for health check and information:

    Sudo su - hdfs

    hdfs dfsadmin -report

  3. Check disk usage for folder /user/cloudera/:

    hdfs dfs -du /user/cloudera/

    Run the same command above but with human readable format.

  4. Create a big file that we will move to HDFS and then use for the next section.

    fallocate -l 1G /home/cloudera/labs/blocks.test

    hdfs dfs -put /home/cloudera/labs/blocks.test /user/cloudera/

  5. Again, run the file system check command to check the new added file:

    hdfs fsck /user/cloudera/blocks.test
